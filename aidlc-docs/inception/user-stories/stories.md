# User Stories

스토리 구성: User Journey 기반 (앱 진입 → 설정 → 기록 → 조회 → 통계)

---

## Epic 1: 초기 설정

### US-1.1: 카테고리 설정
**As a** 가계부 사용자  
**I want to** 소비 카테고리를 등록/수정/삭제할 수 있도록  
**So that** 소비를 분류하여 체계적으로 관리할 수 있다

**Acceptance Criteria**:
- [ ] 기본 카테고리(식비, 교통, 쇼핑, 주거, 통신, 의료, 여가, 기타)가 초기 제공된다
- [ ] 새 카테고리를 이름으로 추가할 수 있다
- [ ] 기존 카테고리 이름을 수정할 수 있다
- [ ] 카테고리를 삭제하면 해당 소비 내역은 "기타"로 변경된다
- [ ] 카테고리 이름이 비어있으면 등록이 거부된다

**예외 시나리오**:
- 중복된 카테고리 이름 입력 시 에러 메시지 표시

---

### US-1.2: 카드 설정
**As a** 가계부 사용자  
**I want to** 사용하는 카드를 등록/수정/삭제할 수 있도록  
**So that** 소비 시 어떤 카드를 사용했는지 추적할 수 있다

**Acceptance Criteria**:
- [ ] 카드 이름을 입력하여 등록할 수 있다
- [ ] 등록된 카드 목록을 조회할 수 있다
- [ ] 카드 이름을 수정할 수 있다
- [ ] 카드를 삭제하면 해당 소비 내역은 "카드 없음"으로 변경된다
- [ ] 카드 이름이 비어있으면 등록이 거부된다

**예외 시나리오**:
- 중복된 카드 이름 입력 시 에러 메시지 표시

---

### US-1.3: 사용자(라벨) 설정
**As a** 가계부 사용자  
**I want to** 가족 구성원 이름을 등록/수정/삭제할 수 있도록  
**So that** 누가 소비했는지 표시할 수 있다

**Acceptance Criteria**:
- [ ] 사용자 이름을 입력하여 등록할 수 있다
- [ ] 등록된 사용자 목록을 조회할 수 있다
- [ ] 사용자 이름을 수정할 수 있다
- [ ] 사용자를 삭제하면 해당 소비 내역은 "사용자 없음"으로 변경된다
- [ ] 사용자 이름이 비어있으면 등록이 거부된다

**예외 시나리오**:
- 중복된 사용자 이름 입력 시 에러 메시지 표시

## Epic 2: 소비 기록

### US-2.1: 소비 내역 등록
**As a** 가계부 사용자  
**I want to** 소비 내역을 날짜, 금액, 사용자, 카드, 카테고리, 메모와 함께 등록할 수 있도록  
**So that** 가족의 소비를 체계적으로 기록할 수 있다

**Acceptance Criteria**:
- [ ] 날짜와 금액은 필수 입력이다
- [ ] 금액은 양수만 허용된다
- [ ] 사용자, 카드, 카테고리는 등록된 목록에서 선택한다 (선택사항)
- [ ] 메모는 자유 텍스트로 입력 가능하다 (선택사항)
- [ ] 등록 성공 시 목록에 즉시 반영된다
- [ ] 여러 건을 연속으로 빠르게 입력할 수 있다 (일괄 기록 시나리오)

**예외 시나리오**:
- 금액에 0 또는 음수 입력 시 에러 메시지 표시
- 날짜 미입력 시 에러 메시지 표시

---

### US-2.2: 소비 내역 조회
**As a** 가계부 사용자  
**I want to** 월별로 소비 내역 목록을 확인할 수 있도록  
**So that** 특정 월의 소비 현황을 파악할 수 있다

**Acceptance Criteria**:
- [ ] 월 선택으로 해당 월의 소비 내역만 표시된다
- [ ] 각 내역에 날짜, 금액, 사용자, 카드, 카테고리, 메모가 표시된다
- [ ] 최신순(날짜 내림차순)으로 기본 정렬된다
- [ ] 카테고리별 필터링이 가능하다
- [ ] 카드별 필터링이 가능하다
- [ ] 사용자별 필터링이 가능하다

**예외 시나리오**:
- 해당 월에 내역이 없으면 빈 상태 메시지 표시

---

### US-2.3: 소비 내역 수정
**As a** 가계부 사용자  
**I want to** 기존 소비 내역을 수정할 수 있도록  
**So that** 잘못 입력한 정보를 바로잡을 수 있다

**Acceptance Criteria**:
- [ ] 기존 내역의 모든 필드(날짜, 금액, 사용자, 카드, 카테고리, 메모)를 수정할 수 있다
- [ ] 수정 시에도 필수 필드 검증이 적용된다
- [ ] 수정 성공 시 목록에 즉시 반영된다

**예외 시나리오**:
- 수정 중 서버 오류 시 에러 메시지 표시, 기존 데이터 유지

---

### US-2.4: 소비 내역 삭제
**As a** 가계부 사용자  
**I want to** 소비 내역을 삭제할 수 있도록  
**So that** 잘못 등록된 내역을 제거할 수 있다

**Acceptance Criteria**:
- [ ] 삭제 버튼 클릭 시 확인 팝업이 표시된다
- [ ] 확인 시 내역이 삭제되고 목록에서 즉시 제거된다
- [ ] 취소 시 아무 동작 없이 팝업이 닫힌다

**예외 시나리오**:
- 삭제 중 서버 오류 시 에러 메시지 표시

---

## Epic 3: 통계 조회

### US-3.1: 월별 통계 조회
**As a** 가계부 사용자  
**I want to** 특정 월의 소비 통계를 카테고리별/카드별/사용자별로 확인할 수 있도록  
**So that** 한 달간의 소비 패턴을 한눈에 파악할 수 있다

**Acceptance Criteria**:
- [ ] 월을 선택하여 해당 월 통계를 조회할 수 있다
- [ ] 카테고리별 총 소비 금액과 비율(%)이 차트로 표시된다
- [ ] 카드별 총 소비 금액과 비율(%)이 표시된다
- [ ] 사용자별 총 소비 금액과 비율(%)이 표시된다
- [ ] 월 총 소비 금액이 표시된다
- [ ] 전월 대비 증감액과 증감률이 표시된다

**예외 시나리오**:
- 해당 월에 데이터가 없으면 "데이터 없음" 메시지 표시
- 전월 데이터가 없으면 증감 비교 항목 미표시

---

### US-3.2: 연간 통계 조회
**As a** 가계부 사용자  
**I want to** 연간 월별 소비 추이를 확인할 수 있도록  
**So that** 1년간 소비가 늘고 있는지 줄고 있는지 파악할 수 있다

**Acceptance Criteria**:
- [ ] 연도를 선택하여 해당 연도 통계를 조회할 수 있다
- [ ] 1~12월 월별 총 소비 금액이 차트(라인/바)로 표시된다
- [ ] 카테고리별 월별 소비 추이를 비교할 수 있다
- [ ] 카드별 월별 소비 추이를 비교할 수 있다
- [ ] 사용자별 월별 소비 추이를 비교할 수 있다
- [ ] 연간 총 소비 금액이 표시된다
- [ ] 전년 대비 증감률이 표시된다

**예외 시나리오**:
- 해당 연도에 데이터가 없으면 "데이터 없음" 메시지 표시
- 전년 데이터가 없으면 증감 비교 항목 미표시

## INVEST Criteria Compliance
- **Independent**: 각 스토리는 독립적으로 구현/테스트 가능
- **Negotiable**: acceptance criteria 내에서 구현 방식 유연
- **Valuable**: 각 스토리가 사용자에게 직접적 가치 제공
- **Estimable**: 명확한 범위로 추정 가능
- **Small**: 단일 기능 단위로 분리
- **Testable**: acceptance criteria로 테스트 가능
